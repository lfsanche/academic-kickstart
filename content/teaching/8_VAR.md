```python
def adfuller_test(series, signif=0.05, name='', verbose=False):
    """Perform ADFuller to test for Stationarity of given series and print report"""
    r = adfuller(series, autolag='AIC')
    output = {'test_statistic':round(r[0], 4), 'pvalue':round(r[1], 4), 'n_lags':round(r[2], 4), 'n_obs':r[3]}
    p_value = output['pvalue']
    def adjust(val, length= 6): return str(val).ljust(length)

    # Print Summary
    print(f'    Augmented Dickey-Fuller Test on "{name}"', "\n   ", '-'*47)
    print(f' Null Hypothesis: Data has unit root. Non-Stationary.')
    print(f' Significance Level    = {signif}')
    print(f' Test Statistic        = {output["test_statistic"]}')
    print(f' No. Lags Chosen       = {output["n_lags"]}')

    for key,val in r[4].items():
        print(f' Critical value {adjust(key)} = {round(val, 3)}')

    if p_value <= signif:
        print(f" => P-Value = {p_value}. Rejecting Null Hypothesis.")
        print(f" => Series is Stationary.")
    else:
        print(f" => P-Value = {p_value}. Weak evidence to reject the Null Hypothesis.")
        print(f" => Series is Non-Stationary.")
```

## Analsis de residuos

En las secciones anteriores, hemos considerado procedimientos para elegir el orden de un modelo VAR para el proceso de generaci√≥n de una serie de tiempo m√∫ltiple dada. Los criterios para la elecci√≥n del modelo pueden considerarse criterios para decidir si los residuos est√°n lo suficientemente cerca del ruido blanco. Por supuesto, si, por ejemplo, el pron√≥stico es el objetivo, puede no ser de suma importancia si los residuos son realmente ruido blanco siempre que el modelo pronostique bien.

Sin embargo, hay situaciones en las que resulta interesante comprobar el supuesto de ruido blanco para los residuos de un modelo en particular. Por ejemplo, si el orden del modelo se elige por m√©todos no estad√≠sticos (por ejemplo, sobre la base de alguna teor√≠a econ√≥mica), puede ser √∫til tener herramientas estad√≠sticas disponibles para investigar las propiedades de los residuos.

# Ho: Residuals are white noise

## Normalidad de residuos

Se necesita la normalidad del proceso de generaci√≥n de datos, por ejemplo, en la configuraci√≥n de intervalos de pron√≥stico. Los residuos no normales tambi√©n pueden indicar de manera m√°s general que el modelo no es una buena representaci√≥n del proceso de generaci√≥n de datos. Por lo tanto, probar este supuesto de distribuci√≥n es deseable.

results.test_normality().summary()
